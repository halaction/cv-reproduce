\section{Ablation Study}

The ablation code path is already prepared in the repository, and the current Weights \& Biases export already contains a first round of ablation results. These runs are useful for understanding the direction of the effects, although some comparisons remain preliminary because not every reference run was stopped at the same epoch.

These ablations are not arbitrary extensions. Each one targets a specific component that is easy to isolate in the ConvMixer design: the patch-forming front-end, the expressiveness of channel mixing, and the dependence of the model on training-set size. Together, they are meant to answer whether ConvMixer works well because of the exact combination proposed in the paper or because a broader family of simple patch-and-convolution hybrids is already strong on CIFAR-10.

\subsection{Patch Stem Replacement}

This subsection studies patch embedding vs.\ a convolutional stem. The ablation replaces the original patch embedding stem with a more conventional convolutional stem that preserves the downstream tensor shape. The goal is to test whether the patch projection itself is an important ingredient of ConvMixer, or whether a standard convolutional front-end can recover most of the same behavior once the later mixing blocks are unchanged.

The current exported run reaches best test accuracy $0.8870$ by epoch 31, which is very close to the base ConvMixer run at $0.8848$. This suggests that replacing the patch stem with a shape-compatible convolutional alternative does not obviously destroy performance in the current setup. At the same time, both the base ConvMixer run and this ablation were interrupted early, so the result should still be treated as preliminary.

\subsection{Non-Linear Channel Mixing}

This subsection studies pointwise channel mixing vs.\ non-linear channel mixing. The base ConvMixer block uses a single $1 \times 1$ convolution for channel mixing. The non-linear channel mixing ablation replaces that stage with a deeper non-linear mixer while keeping the rest of the block structure fixed. This tests whether the original pointwise mixer is sufficient, or whether additional channel-wise capacity materially changes performance.

In the current export, this is the strongest ConvMixer-family run, reaching best test accuracy $0.9125$ after 50 epochs. On its face, this suggests that a deeper non-linear channel mixer can outperform the original pointwise convolution. The comparison is not perfectly controlled because the base ConvMixer run stopped much earlier, but the gain is large enough to justify further investigation.

\subsection{No Channel Mixing}

This subsection studies pointwise channel mixing vs.\ no channel mixing. The ablation removes the channel mixing stage entirely by replacing it with an identity mapping. It is the cleanest way to test whether the pointwise mixing path is essential, because the spatial depthwise branch and the classifier head remain intact.

This ablation shows a large drop, reaching only $0.7421$ best test accuracy after 50 epochs. The difference from both the base model and the non-linear channel variant strongly supports the claim that channel mixing is essential: spatial depthwise processing alone is not enough to recover ConvMixer performance.

\subsection{Data-Size Robustness}

This ablation evaluates all three model families with 50\%, 10\%, and 1\% of the CIFAR-10 training set, while keeping the full test set. The resulting trends are shown in Figure~\ref{fig:ablations}. As expected, all models degrade as data becomes scarce. At 50\% data, ConvMixer reaches $0.8407$, ResNet18 reaches $0.7896$, and DeiT-Tiny reaches $0.5401$. At 10\%, the ordering remains the same: ConvMixer reaches $0.6637$, ResNet18 reaches $0.6466$, and DeiT-Tiny reaches $0.3965$. At 1\%, the models collapse further, but ConvMixer still leads with $0.4077$, followed by ResNet18 at $0.3760$ and DeiT-Tiny at $0.3000$.

The small-data results are therefore consistent with the main experiment ranking rather than overturning it. ConvMixer remains the strongest model across all tested fractions, and ResNet18 remains second. DeiT-Tiny shows some early movement in the curve visualizations, but by the end of training it is still the weakest model at every data level in the current setup.
